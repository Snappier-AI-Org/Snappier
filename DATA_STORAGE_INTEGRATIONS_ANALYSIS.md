## Notes

- **Open source** means the software is free and open source
- **Self-hosted** means you can run it on your own server for free
- **Free tier** means the service offers a free plan with API access
- **No business verification** means you can sign up with a personal account
- **Standard protocol** (like FTP, GraphQL) means it's a protocol/format, not a service
- **Database connections** typically require connection strings/credentials
- **Cloud storage** integrations usually require API keys

## Special Considerations

### Database Integrations
- Require connection strings (host, port, database, username, password)
- May need SSL/TLS configuration
- Consider connection pooling for production
- Support for both read and write operations

### Cloud Storage Integrations
- Require API keys or OAuth
- Support for file upload, download, list, delete operations
- May need to handle large file uploads
- Consider webhook triggers for file changes

### Database-as-a-Service
- Usually REST API based
- May have rate limits on free tiers
- Webhook support for real-time updates
- Good documentation typically available
